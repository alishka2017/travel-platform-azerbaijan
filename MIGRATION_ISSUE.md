# Проблема с миграцией данных в Sanity

## Обнаруженная проблема

При попытке запуска миграции данных получена ошибка:

```
{"error":{"description":"project user not found for user ID \"g-8TwMLgdWP2sn\" in project \"mk35mbbk\""}}
```

## Причина

Токен `SANITY_API_TOKEN` принадлежит роботу `g-8TwMLgdWP2sn` (Hermes agent (Robot)), но этот робот **не добавлен как участник проекта** `mk35mbbk` (Smartholiday.az).

Проверка участников проекта показала:
- В проекте есть только 1 участник: `p3kw3LNNE` (administrator)
- Робот `g-8TwMLgdWP2sn` отсутствует в списке участников

## Решение

Нужно добавить робота в проект через веб-интерфейс Sanity:

1. Зайти в [Sanity.io](https://www.sanity.io)
2. Выбрать проект `mk35mbbk` (Smartholiday.az)
3. Перейти в **Settings** → **Members**
4. Добавить пользователя с ID `g-8TwMLgdWP2sn` (Hermes agent (Robot))
5. Назначить роль **Editor** (редактор)
6. Запустить миграцию снова

## Альтернативное решение

Если добавление робота невозможно, можно:

1. Создать новый токен от имени администратора (`p3kw3LNNE`)
2. Обновить `.env.local` с новым токеном
3. Запустить миграцию

## Проверка токена

Токен валиден и имеет доступ к проекту:
- Токен начинается с `skDzDxl...`
- Длина: 180 символов
- Проект `mk35mbbk` существует и активен
- Проблема только в правах доступа робота
